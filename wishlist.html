Want to help with SBTCVM?

Notes:

  • SBTCVM is written and python, and uses pygame for graphics, sound, user input, etc. knowledge of these helps.
  • SBTCVM is a balanced ternary VM (obviously), so a basic understanding of balanced ternary helps a lot.
  • a basic understanding of digital computers helps; however, binary digital computers and balanced ternary digital computers do have many differences.
  • in general, expert knowledge in balanced ternary, mathematics, computer technology, or other related fields is greatly appreciated and needed in some cases.

reasonably simple:

  • GUI calculator using libbaltcalc, and preferably pygame (to avoid additional dependencies.)

complicated:

  • Proper, framebuffered display with various advanced features (Simple Balanced Ternary Graphics Adapter (SBTGA))
  • better TTY. (preferably with 3-trit RGB color support, and more intelligent formatting functionality.)
  • mouse support (as a VM device. support for mouse in menus would likely also help)
  • proper keyboard support
  • Virtual Digital audio device. (SBTDAC)
  • Memory viewer.
  • more advanced assembly support. (i.e. library/module support, etc.)
  • memory range copying. (ability to copy large sections of memory more easily)

extra complicated:

  • Fully working OS. (SBTC-OS)
  • Well designed, portable, balanced ternary programming language, with library/module support. (SBTPL)
  • Virtual Filesystem support of some kind
  • Virtual removable "disks" with filesystem, and related external tools.
  • Balanced ternary floating point support in libbaltcalc. (perhaps offer both true balanced ternary operations, and faster approximate operations) (main challenge will be the high-speed conversion functions.)
  • Balanced ternary fixed point arithmetic support in libbaltcalc. (main goal here is exact accuracy, not speed.)
  • DMA support for IObus devices. useful for SBTDAC and SBTGA especially.
  • move IObus device handling to a concurrent thread. (such as the TTY, user input, SBTDAC and  SBTGA)

Research:

  • come up with a means of storing balanced ternary floating points using ternary data.
  • look into further what work has been done in the past.
  • plan out basic structure of SBTC-OS. perhaps look at various alternate, simple Operating Systems for inspiration.
  • general hardware research.
  • research various kinds of computer operations and functionality, and how these may translate into SBTCVM and Balanced Ternary Computers in general.
  • look into what kinds of software, functions, and algorithms, SBTCVM can run now, or may be able to run in the future. perhaps libbaltcalc may help?

Other:

  • Help Spread the word of SBTCVM, and help others in learning SBTCVM, tasm, and balanced ternary in general.
  • Proper website. with wiki, and forum. (please contact the dev team if you want to help with this!)
  • IRC channel (ideally on freenode) (please contact the dev team if you want to help with this!)

Completed (implemented):

  • pointer addressing / c arrays (ability to read and write memory locations by a pointer the program can increment)
